인터넷이라는 것은 전 세계에 있는 컴퓨터들의 정보를 모아놓은 것입니다.그 중에 대부분의 정보는 WWW (월드 와이드 웹)으로 접근할 수 있습니다.한 페이지를 받아서,읽고 바라보고 원하면 다른 페이지로 접속할 기능이 있습니다.

어떤 페이지들은 깜짝 놀랄 정도로 화려하기도 합니다.이런 페이지들은 이용자가 읽기 쉽게 정리되어 있고,다양한 소리와 사운드가 가능합니다.그러나 어떤 페이지들은 기대와는 완전히 반대일 수 있습니다.그러므로,이용자의 목적은 자신에게 중요하거나 필요한 페이지들만 골라내는 기술을 익히는 것입니다.

WWW 에서 제일 중요한 특징은 그래픽이나 사운드가 아니고 내용입니다.한 페이지에 모든 내용을 다 담을 수 는 없기 때문에 원하는 내용을 가지고 있는 페이지의 주소나 그 페이지로 접속해 주는 Link가 필요합니다. 어떤 페이지들은 다른 페이지들에게 자동적으로 접속해 주는 Link 가 작성되어 있기도 합니다.이런 Link (Highlight나 밑줄된 단어/단문/이미지)는 마우스로 클릭하기만하면 다른 페이지를 화면에 읽어 들입니다. 아주 많고 다양한 페이지들이 Link로 연결되 있을 수도 있습니다.그러므로,원하는 정보를 모두 한 페이지에서 찾는다고 생각하지 말고, 많은 페이지에서 조금씩 조금씩 모으려고 생각하십시오.

Netscape는 WWW 페이지를 풍부하게 진열하고 원하는 페이지를 쉽게 찾아서 받아 들이는 기능 때문에 인기가 있습니다.  Netscape를 이용하면 인터넷의 text-based(텍스트) 서비스만 이용하는 것이 아니라,멀티미디어로 가득한 WWW 페이지도 사용할 수 있습니다.
WWW 페이지란 무엇인가?


인터넷 WWW 정보는 컴퓨터 화면에서 페이지 포맷으로 접근할 수 있는데 사용방법도 이해하기 쉽습니다.
아무 페이지에서 시작합니다 (이때 처음 불러오는 페이지를 홈페이지라고 합니다)

Link (Highlight나 밑줄된 단어/이미지)를 마우스로 선택하면 다른 장소로 연결시켜 줍니다.

메뉴바 (Menu Bar)의 화살표들을 이용해서 이미 본 페이지로 돌아갑니다.또, 메뉴 아이템을 선택해서 이용자가 관심 있는 페이지에 직접 접근할 수가 있습니다.

이미 본 페이지들은 Go 메뉴에 기록되었으므로, 아무때나 돌아갈 수 있습니다.

가끔 접속하거나,볼만하다고 생각되는 페이지들은 Bookmark 메뉴에 저장해서, 다른 때에 불러오고 싶을 경우에 직접 접속할 수 있습니다.

Directory 메뉴에 인터넷의 특징들을 설명해 주는 페이지들을 기록해 줍니다.



잡지를 읽는 것과 같이, 많은 페이지들 사이를 왔다갔다하고 싶을 때가 있을 것입니다.- 같은 기사를 계속 읽고 싶을때, 또는 다른 기사를 새로 읽으려고 시작하려고할 때.그러나, 잡지의 종이 페이지같이 많은 WWW 페이지를 한 손에 갖고 있을 수는 없습니다.WWW 페이지들는 각각 길이도 틀리고 하나씩 하나씩 따로 보아야 하므로,정보가 어디에서 시작하고 어디에서 끝나는지 알기가 어려울 수도 있습니다.

그렇기 때문에,인터넷이 많은 정보를 효과적으로 제공하기 위하여 조력하고 있음에도 불구하고,많은 페이지 사이에 퍼져 있는 정보를 모아 사용하려면 불편할 때도 있을 것입니다.인터넷을 처음 사용하는 분들은 끝이 없다고 느끼는 경우도 있을 것입니다.

그래서 인터넷 이용자들은 많은 페이지들을 관리하는 도구가 필요합니다.Netscape의 text-fields (정보입력항목), toolbar 버튼들과 메뉴 아이템들이 많은 정보를 관리해주고 있습니다.

모든 페이지가 유일한 URL이 있습니다.인터넷만큼 큰 네트워크에서 어느한 페이지가 다른 페이지들과 어떻게 구별되는지 알려면, 우선적으로 URL (Uniform Resource Locator)이 무엇인가 알아야 됩니다.  사람마다 유일한 손자국이 있는 것처럼, 페이지마다 유일한 URL이 있습니다.

URL은 컴퓨터 네트워크에 있는 파일(페이지, 이미지,...)의 성격과 주소를 정하는 문서입니다.한마디로 URL은 페이지의 공간 정보이고, Netscape는 이 URL을 Location Field에다 진열합니다.보통 경우에는 정확한 URL을 알 필요가 없습니다.  왜냐하면 페이지의 URL은 Link에 보통 정해지기 때문입니다;Netscape의 Link, Toolbar 아이콘이나 메뉴 아이템을 선택할 때는 URL이 자동적으로 컴퓨터에 알립니다. 그러나 가끔 원하는 페이지에 연결하는 Link가 없고 URL문서만 있을 때도 있습니다.(친구들사이에서 또는 신문광고에서)

Netscape에서 URL을 직접 입력하려면 Tool Bar의 Open 아이콘이나 File 메뉴의 Open Location 아이템을 선택하면 됩니다.  이 URL을 가지고 Netscape가 Link를 따라가는 것처럼 그 페이지에 접근합니다.
URL의 예:http://home.netscape.com./index.html


Netscape가 URL을 가지고 인터넷에 접속된 모든 컴퓨터중에서 선택된 한 아이템을 찾습니다.URL 문서안에 Protocol/Server하고 아이템의 경로명을 정하는 부분이 있습니다.
프로토콜 후 ":"가 있고 (http:), Server 전에는 "//" 가 있고 (//home.netscape.com),경로명부분들사이에 "/"가 있습니다.(/index.html)

URL의 첫 부분 - 프로토콜 - 은 받은 정보를 기계번역하는 방식을 정합니다. 많은 인터넷 페이지들은 HTTP(Hypertext Transfer Protocol)를 씁니다.  또, 다른 프로토콜 종류에서 가끔 보이는 것들은 File (아니면 FTP: File Transfer Protocol), news (newsgroup)와 gopher)

두번째 부분 - 서버 - 는 이용자가 원하는 정보가 저장된 컴퓨터를 동일합니다(identify).  인터넷의 서버들은 보통 그 컴퓨터를 관리하는 회사나 조직의 이름입니다.

마지막 부분 - Pathname - 은 정보가 저장된 컴퓨터를 접속해서 원하는 파일이나 파일들의 위치를 동일합니다.  예를 들면 파일명 "/welcome.html" 만 있을 수도 있고, 아니면 많은 파일이 있는 디렉토리 일수도 있습니다.

Link를 찾고, 선택하고, 취소함


Link는 어떠한 페이지를 다른 페이지와 연결시켜 주는 것입니다.페이지 내용 중에서 Highlight나 밑줄친 단어들은 Link라는 뜻입니다.Highlight된 아이콘이나 이미지들도 Link와 같은 기능을 합니다.마우스의 커서가 Link를 표시할 때는, 그 Link의 URL이 밑의 Status Field에 나타납니다.

Link가 연결하는 페이지를 받아들이면 Highlight된 단어, 이미지나 아이콘을 한번 눌러 주십시오.Link를 선택함으로써 정보를 서버에서 이용자의 컴퓨터로 옮깁니다.선택하면, 위-오른쪽에 있는 Netscape Status Indicator가 페이지를 옮기는 중임을 나타낼 것입니다.전송 중, 아무때나 Stop 아이콘이나 Go 메뉴의 Stop Loading 명령을 선택하면 전송이 취소됩니다.

이미 선택된 Link와 아직 선택 안된 Link들은 다른 색깔로 진열됩니다.흑백모니터에서는 Link들이 밑줄밖에 안되 있어서, 선택된 Link와 안된 Link와 차이가 없습니다. 마우스가 어떤 Link를 표시할 때는 그 Link가 연결시켜 주는 페이지의 URL이 Status Message Text Field (화면의 밑-왼쪽)에 나타납니다.아직 선택 안된 Link들은 파랑 색으로 나타나고,이미 선택된 Link들은 자주색입니다. (Windows/Macintosh 사용자들은 Options 메뉴로 색깔들을 바꿀 수 있고, UNIX 사용자들은 응용 프로그램 밖에서 바꿀 수 있습니다)

전송중에 취소하는 명령이 가끔씩은 유리합니다.특히, 페이지 전송이 너무 오래 걸린다고 생각 하실 때는 요긴합니다.   페이지 내용이 많거나, 그 서버를 접속하는 사람들이 많을 때는 전송속도가 느려져서 시간이 많이 걸릴 수 있습니다.  또는 접속하고 싶은 서버나 파일이 유효하지 않을 때도 있습니다.이런 경우에는,접속을 못했거나 파일을 못 찾겠다는 메시지가 나타납니다.

새 페이지를 받아들일 때, 페이지 전체가 나타나거나,내용이 많은 경우에는 한 부분씩 나타날 것입니다.화면에 처음 나타날 때는 보통 그 페이지의 첫 줄에서 시작하지만, 때마다 그 페이지의 특별한 부분으로 직접 연결할 경우도 있습니다. 때로는, 새 페이지로 가지 않고, 같은 페이지의 다른 부분으로 갑니다.
아이콘과 메뉴 아이템으로 Link 함


페이지 내용안에 있는 Link들 말고도, Netscape 아이콘 버튼이나 메뉴 명령으로 다른 페이지로 연결하는 기능도 있습니다.  이런 종류의 명령들은 대부분 벌써 연결했었던 페이지들을 불러오는 것입니다.  버튼명령들은 잠시 전에 읽었던 페이지 사이에서 왔다갔다 할때 특히 편합니다. 메뉴 아이템을 통해서는 다양한 페이지들에게 직접 연결할 수 있습니다. Go 메뉴 밑에는 이미 방문했던 페이지의 제목들이 목록으로 저장되어 있습니다 (이것을 History List라고 합니다).  Bookmark 메뉴에는 사용자의 입맛에 맞는 페이지들이 저장되어 있습니다.

Link 명령버튼과 메뉴 아이템들은 모두 URL을 거치지 않고 직접 연결시켜 줍니다.그러나, 원하는 장소에 연결 해주는 Link가 없을 때는 URL을 직접 Location Field에 입력할 수 있습니다; Toolbar의 Open 버튼이나 File 메뉴의 Open Location 아이템을 선택하면, URL을 입력할 수 있는 Dialog Box 가 나타납니다.
  
Toolbar의 많이 쓰는 버튼들은 다음과 같습니다
Back:History List에 현재 페이지의 바로 앞에 있는 페이지를 불러옵니다.

Forward:History List에 현재 페이지의 바로 뒤에 있는 페이지를 불러옵니다.

Home: Options메뉴의 Preferences Box에 입력된 Homepage 를 불러옵니다.

Open: Dialog Box를 불러와서 사용자가 이미 아는 URL을 입력합니다.



메뉴 아이템들은 Toolbar의 명령 버튼들보다 더 많은 옵션들을 제공합니다.
Go메뉴는 이미 본 페이지들을 순서대로 출력합니다.제일 최근의 페이지가 목록 제일위에 있습니다.

Bookmark 메뉴에는 이용자가 자신에게 중요하여 저장한 페이지들을 기록합니다.언제나,목록된 제목들중 하나를 선택하면 그 페이지로 직접 연결합니다.기존목록에 현재 페이지를 기록할 때는 Bookmark메뉴의 Add Bookmark를 선택 하면 됩니다.View Bookmarks를 선택하면 이미 저장된 아이템들의 속성을 변경해 주는 Dialog Box를 불러옵니다.

Directory와 Help메뉴들은 Netscape 와 인터넷을 안내하는 페이지들을 목록으로 제공합니다.



Pop-up 메뉴는 유틸리티 옵션들(Utility Features)과 어떤 Links의 단축키를(Shortcut) 제공합니다. Windows와 UNIX 이용자들은 Pop-up메뉴를 불러오기 위하여 마우스의 오른쪽 버튼을 누르면 됩니다.(Macintosh 이용자들은 마우스 버튼을 계속 누르면 메뉴가 나옵니다) 마우스가 Link나 이미지를 표시하면서 Pop-up 메뉴를 불러오면 많은 명령을 제공합니다: 페이지 불러오기, 이미지 하나씩 보기, 파일 복사/저장...
History와 Bookmark 목록 사용 방법


History와 Bookmark의 아이템들은 이용자의 행동(원하는 페이지들을 선택하는 것)에 따라서 만들어지기 때문에 특별히 신경쓸 필요가 있습니다.
  
History와 Bookmark아이템들은 제목목록으로 정리되어 있습니다.페이지를 불러오려면, 메뉴를 열어서 원하는 페이지의 제목을 선택하십시오.  목록이 길고 제목이 페이지의 내용을 충분히 기술하지 않을 경우에는, 원하는 페이지를 찾는 것이 생각외로 어려울 수도 있습니다.
  
Go메뉴의 History아이템들은 하나씩 순서대로 목록화되어 있습니다.예를 들면 순서대로 '아시아', '한국', '경기도'그리고 '서울' 이라는 페이지들을 연결하면, History List에는 4개의 아이템들이 모두 기록되어 있을 것입니다.('서울'이 제일 위에) 그러나 그 다음에 다시 한국 페이지로 돌아갔다면, History List에는 '아시아'와 '한국'밖에 없을 것입니다.  처음 페이지(홈 페이지)에서 현재 페이지의 지름길만 기록되어 있을 것입니다.

Bookmark메뉴는 Go메뉴와는 달리 별도의 데이터 베이스체제로 기록하였다가 불러오는 방법입니다.  Go메뉴의 아이템들은 Netscape를 종료할 때마다 삭제되지만 Bookmark의 아이템들은 이용자가 개인적으로 삭제하거나 모든 목록을 바꿀 때까지는 계속 있을 것입니다.  Netscape는 Bookmark목록을 만들 때 많은 옵션을 제공합니다.  기본적인 옵션은 물론, 페이지의 제목을 목록에다 삽입하는 것이지만, 더 진보적인 옵션들도 제공합니다.

Bookmark메뉴의 View Bookmarks 아이템을 선택하면 "Bookmark List" Dialog Box가 나타납니다.이 Dialog Box가 제공하는 명령 버튼을 갖고 존재하는 목록 아이템들을 관리하는 기능이 있습니다.  이 진보적인 옵션들이 약간 복잡하다고 생각하셔도, 낙담할 필요는 없습니다.  기본적인 옵션으로 간단하게 사용하기만 해도, 매우 이용가치가 많은 도구 입니다.


자신의 취향에 따라 Netscape 프로그램의 환경을 설정할 수 있습니다.기본적인 옵션들은 Option 메뉴에 있습니다.

Netscape는 기본적으로 Toolbar, Location Field와 Directory (디렉토리) 버튼들이 화면에 진열되게 설정됐습니다.  이런 그래픽(Graphical Element) 들은 이용자에게 도움이 되라고 선택되었지만 쉽게 없앨 수 있습니다.이런 그래픽을 숨김으로써 한 화면에서 한번에 읽을 수 있는 내용이  증가합니다.
Option 메뉴의 Show Toolbar아이템을 선택하면 메뉴들중에서 특히 많이 쓰이는 명령들이 아이콘 버튼으로 화면 위에 나타납니다.

Option 메뉴의 Show Location아이템을 선택하면 Location Field와 현재 페이지의 URL이 진열됩니다.

Option 메뉴의 Show Directory Buttons아이템을 선택하면 Directory와 Help메뉴중에서 가끔 쓰이는 명령들이 화면에 아이콘 버튼들로 표시됩니다.



Option 메뉴의 남은 아이템들 - Show FTP File Information 과 Auto-Load Images - 들은 다른 아이템들 같이 화면의 모습을 변형하지 않습니다.  Show FTP File Information는 FTP 포맷으로 받는 정보를 제대로 기계번역하고, Auto Load Images는 새 파일을 받으면서 내용중에 있는 이미지들을 자동적으로 풀어주는 옵션입니다.

Option메뉴의 Preferences아이템을 선택하면, 많은 Sub-menu들을 제공하는 Dialog Box가 진열됩니다: Windows Styles, Link Style, Font, Color, Mail, News, Cache, Network, Applications, Directories, Images, Security, Proxies하고 Helper Applications.

각 Sub-menu는 Netscape가 어떻게 작동되는지 그 조종하는 명령들을 제공합니다.  대부분의 명령들은 Netscape 화면의 출현을 조종합니다.  환경을 변경한 후, 실행시키려면 OK 버튼을 선택하고 취소하시려면 Cancel 버튼을 선택하십시오.

스타일,글꼴과 색상변경


스타일, 글꼴과 칼라를 변경할 경우에는 Options메뉴의 Preferences아이템을 선택해서, 진열되는 Dialog Box에서 스타일, 글꼴이나 칼라 메뉴를 선택하십시오.

Window Style의 메뉴를 선택하면 몇몇의 둥근 선택 버튼들이 진열됩니다.이 버튼들은 Netscape의 Toolbar가 아이콘이나, 텍스트로만 나오든지 아니면 아이콘과 텍스트가 같이 나올 것인지를 조종합니다.

Link Style 메뉴를 선택하면 Check Box가 진열될 것입니다.이 옵션을 선택하면 모든 Link들이 밑줄이 처집니다.흑백 모니터에서는, Link들을 보통 텍스트와 구별할 수 있기 위해 이 옵션이 항상 선택 되 있어야 합니다.

Link Styles에는 Check Box만 아니라, 따로 있는 라디오 버튼들도 있습니다.이 버튼들은 선택된 Link들이 언제까지 자주색으로 남을 것인지를 관리합니다.예를 들면, 7 일로 입력되었으면 어떤 Link를 선택한 후 7일 이후 그 Link는 다시 보통 색깔로 변합니다.Never를 선택하면 기간에 상관없이 계속 자주색이 되고,Now를 선택하면 모든 Link들은 즉시 보통 색깔로 변합니다.Netscape는 선택만으로 30일로 설정되었습니다.

Windows 와 Macintosh 이용자들은 Link의 색깔을 설정할 때는 Preferences의 Color메뉴를 선택하십시오.Link의 보통 컬러와 선택된 컬러를 따로 정해야 합니다.

Windows와 UNIX 이용자들은 Images메뉴를 선택해서 받는 이미지의 칼라와 이용자의 컴퓨터가 제공하는 칼라와 맞추는 기능도 있습니다.

Fonts 메뉴에서는 페이지의 텍스트의 글꼴의 두 종류 - Proportional와 Fixed - 를 설정할 수 있습니다.  페이지의 대부분의 텍스트는 Proportional 종류의 텍스트입니다.  Fixed Font는 페이지 안에 특별히 정보입력항목이 있을 경우에만 사용됩니다.

Proportional과 Fixed Font 의 선택은 pull-down 메뉴로 진열됐지만, 제공되는 선택들이 위 항목의 For the Encoding (언어) 선택에 따라서 틀립니다.  그러므로, 원하는 For the Encoding 언어를 선택한 후 Proportional와 Fixed의글꼴을 설정하는 게 낫습니다.  태만으로, Netscape의 글꼴들은 이렇게 설정됐습니다: Latin1(For the Encoding), Times New Roman (Proportional), Courier (Fixed).


홈 페이지라는 것은 Netscape를 실행할 때, 화면에 처음 나타나는 페이지입니다.  Netscape 프로그램을 처음 사용할 때도 아마 홈 페이지가  벌써 설정됐을 것입니다.그러나, 홈 페이지는 어떤 페이지로도 개인적으로 설정할 수 있습니다. 그러기 위해서는 Option 메뉴의 Preferences 아이템을 선택한 다음 진열되어 있는 Dialog Box에서 Windows Style을 선택하십시오.Home Page Location의 라디오 버튼과 정보입력항목이 보일 것입니다.선택버튼을 선택한 후 그 항목에 원하는 홈 페이지의  URL을 입력하십시오.특별히 원하는 홈 페이지가 없을 경우에는 Home Page Location 버튼 옆에 있는 Blank Page 버튼을 선택하시면, Netscape를 실행할 때 작동하는 페이지가 없을 것입니다.


이미지와 다른 멀티미디어가 화면에 텍스트처럼 쉽고 빨리 나타난다면 얼마나 좋겠습니까? 그러나 이미지, 사운드와 영화 파일들은 텍스트보다 워낙 (용량이) 크기 때문에 (bytes) 서버에서 이용자의 컴퓨터로 전송하기에는 시간이 더 걸립니다.  전송 시간은 많은 요인에 영향을 받지만 제일 중요한 요인은 모뎀이나 포트의 속도입니다.혹시 너무 느리게 받아들일 경우를 위해서는 Netscape가 이미지들을 조작하는 옵션들을 제공합니다.

Netscape는 페이지의 이미지들을 자동으로 풀게 설정되어 있습니다.그러나,원하면 이 옵션을 끌 수 있습니다.이런 경우에는 Option메뉴의 Auto-Load Images를 선택하십시오.(이 옵션이 켜 있으면 단어 옆에 표시가 있을 것입니다. 다시 선택하면 옵션이 꺼지고 표시가 없어집니다) Auto-Load가 꺼져 있을 때는 이미지 대신에 작은 아이콘이 나타납니다.또, 이런 아이콘 옆에 몇개의 단어도 같이 진열됩니다.이 단어들을 rnative Text (text)라고 합니다.

마음이 바뀌어, 페이지의 이미지들을 보고 싶다면 Toolbar의 Images아이콘이나 View메뉴의 Load Images를 선택할 필요가 있습니다. 많은 이미지중에서 하나만 보고 싶다면은 그 이미지를 마우스로 클릭하십시오.

Auto-Load Images 옵션을 끄는 이점은 전송할 바이트가 더 작아서 더 빨리 받아들일 수 있다는 것입니다. 불이익은 개인적으로 이미지를 진열하라고 명령할 때까지 못 보는 것입니다.

Auto-Load Images 옵션을 선택하면 그 즉시 화면의 변함은 없습니다.  옵션을 변형한 후 받아들이는 페이지에만 영향을 미칩니다.  현재의 페이지에도 적용 하려면 Toolbar의 Reload 아이콘이나 View메뉴의 Reload 아이템을 선택하십시오. 똑같은 페이지를 다시 받는데, 이번에는 옵션이 적용 됩니다.

이용자가 이미지를 받는 방법도 조종 할 수 있습니다.  Options메뉴의 Preferences/Images 아이템을 선택하면은 두개의 선택 버튼이(While Loading과 After Loading) 진열됩니다.  While Loading이 선택됐으면 이미지를 받는 대로 족음씩족음씩 화면에 올립니다.  After Loading은 파일 전체를 받은 후, 모든 이미질를 한꺼번에 올립니다.
External 이미지


External 이미지는 in-line 이미지와 같이 페이지의 내용중에 있지 않고, 다른 창에 따로 진열됩니다.  이런 이미지를  보려면 그 이미지로 연결시켜 주는 Link를 선택하십시오.  Netscape는 GIF, JPEG, 와 XBM포맷의 이미지들을 스스로 풀수 있습니다.  다른 포맷으로 된 이미지들은 적당한 프로그램과 연결시켜야 합니다. Options 메뉴의 Preferences를 선택해서 연결하십시오.

External 이미지들은 다른 페이지들과 같이 유일한 URL이 있고, 이미지의 Link 도 보통 페이지의 Link 와 똑같습니다.  External 이미지들은 Auto-Load Images옵션이 꺼져 있어도 자동적으로 화면에 올려집니다.

External 이미지가 특별히 클 경우에는 그 이미지의 Link 자체가 Snapshot/Thumbnail 이미지 일수도 있습니다.  이런 이미지들은 단어 Link와 똑같이 사용되지만 출현은 진짜 이미지를 아이콘 크기로 줄여서 진열하는 것입니다.  그러므로, 큰 파일을 받기 전에 이미 그 이미지의 출현을 대충 알 수 있습니다.  이 Snapshot이미지를 보시고, 더 자세히 보고 싶다고 생각하시면 그 Link (Snapshot)를 선택하십시오.


Netscape이용자들은 WWW 페이지들을 보고 읽는 것 뿐만이 아닙니다 - 페이지에다 정보를 직접 입력하고 정보를 송신할 기능도 있습니다.  어떤 페이지에는 서류가 있으니 이용자가 정보를 입력해서 그 페이지의 작가에게 보낼 수 있습니다.  이용자가 입력하는 정보는 컴퓨터 화면에 나오지만, 서버의 독창적인 파일은 변경하지 않기 때문에 다른 이용자들은 똑같은 페이지를 받아 들여도 당신이 입력한 정보는 진열이 안됩니다.  서류들은 이용자를 작가와 편하고도 쉽게 통해시켜주는 도구입니다.

보통 전자우편을 보내기 위해서는 수신자의 주소, 편지의 제목과 편지의 내용을 입력해야 합니다.  물론, 서류를 보내기 전에는 정보를 서류에 입력해야 합니다.  그러나 수신자의 주소와 제목은 명시 할 필요는 없습니다 (자동적으로 되어있습니다).


Netscape는 모든 이용자들에게 정보를 안전히 전송할 기능을 제공합니다.  인터넷에서는 정보가 송신하는 컴퓨터에서 받는 컴퓨터로 직접 전송되지 않고, 중간을 통해서 전해지는 컴퓨터들이 많아서, 다른 사람이 몰래 도청할 가능성도 있습니다.

정보가 다른 컴퓨터들 사이에 돌아다니다가, 혹시 정보가 노츨되거나 정보가 복사될 가능이 있습니다.  그래서, 비밀 번호나 Credit Card번호 같은 중요한 정보들이 악용 될 위험이 있습니다.

그러므로, Netscape는 이용자들이 걱정 없이 안전하게 사용할 수 있도록 암호화 기술(patented RSA public key cryptographic technology)을 씁니다.  이용자의 컴퓨터와 접속된 서버만 정보를 암호화(encrypt)하고 번역(decrypt)할 수 있게 되어있습니다.  중간에 누가 복사했다고 해도, 정보는 암호화(encrypt) 됐기 때문에 읽을 수는 없습니다.

모든 서버들은 안전(secure)하지 않습니다. Netscape는 안전(secure)하나 안전하지 않은(insecure)서버와 접속 되있느지는 그래픽(graphical elements)으로 알려 줍니다.
https//로 시작하는 URL의 (http:// 대신) 페이지들은 보안기능이 안전한 서버에서 정보가 온다는 뜻입니다.마찬가지로,snews로 시작되는 (news: 대신)  기사는 안전한 newsgroup에서 온다는 뜻입니다.

Status Bar의 왼쪽에 있는 열쇠 아이콘이 깨져 있으면 파일이 안전하지 않고, 안 깨져있으면 안전한 파일입니다.

Netscape화면의 위부분이 회색이면 안전하지 않고 파랑색이면 안전하다는 뜻입니다.

Option메뉴의  Preferences아이템을 선택해서 안전(Security)에 대한 옵션들을 설정할 수 있습니다.



파일의 안전에 대해서 더 알고 싶은 경우에는 File메뉴의 Document Information 아이템을 선택하십시오.현재 파일의 안전정보가 표시될 것입니다.
뉴스와 전자우편 (e-mail) 서비스에 접근하기


현재 사용하고 있는 뉴스서버와 e-mail서버의 이름을 아십니까?  모르시면 서비스 운영자나 시스템 관리자에게 알아볼 필요가 있습니다.이 서버들의 이름들을 Preferences 아이템으로 입력하면 Netscape가 Newsreader와 e-mail 기능을 제공합니다.

WWW페이지들은 인터넷에서 정보를 접근하는 한 방법입니다.  똑같이 중요한 다른 두 종류는 Usenet Newsgroup와 E-mail 입니다.  그러나 Newsgroup과 e-mail은 다른 프로토콜로 전송하기 때문에, 사용하기 전에 우선적으로 Netscape을 그 서버들과 접속하는 방식을 명시해야 합니다.
  
Newsgroup서버의 이름을 Netscape에 명시할 때, Preferences/E-mail메뉴의 Preferences/News 아이템을 선택해서, 진열되는 News (NNTP) Server Field에 알맞는 이름을 선택하면 됩니다.  지방의 (Local)서버가 있을 경우에는, 그 서버를 입력하는 것이 좋습니다.

마찬가지로, e-mail서버의 이름을 명시할때도 Preferences/email을 선택해서 진열되어있는 Mail (SMPT) Server Box에 입력하기만하면 됩니다.  지방의 (Local) 서버가 있을 경우에는, 그 서버를 선택하는 것이 좋습니다.
Usenet의 기사들을 읽음


Netscape 프로그램은 Newsgroup Posting을 받고 올리는 기능을 제공합니다. Posting이라는 것은 인터넷의 모든 이용자들이 읽을 수 있는 기사 (뉴스) 입니다.  뉴스 Posting을 읽으면서 Netscape가 내용 중에 명령 버튼들을 몇게 제공합니다.  이 버튼들로 기사를 찾거나, 읽은 기사에 대답 하거나, 새 기사를 올릴 수 있습니다.

Usenet는 인터넷의 이용자들이 올리는 가사들을 다양한 Newsgroup으로 정리합니다.  각 Newsgroup은 같은 주제(Subject)의 기사들을 함유 하고 그 주제에 관해서 적당히 기술한 제목이 있습니다.
  
Usenet의 Posting을 접속하는 방법은 보통 페이지를 접속하는 것과 같습니다.  Usenet로 연결시켜 주는 Link를 선택하면 Newsgroup 목록이 진열됩니다.  그 중에서 한 Newsgroup을 선택하고, 또 그 안에 있는 Posting중에 하나를 선택하면, 기사의 내용이 화면에 진열됩니다.  읽고 싶은 Newsgroup이나 Posting의 URL을 이미 알고 있을때는 File/Open Location아이템이나 Open 아이콘으로 직접 접속할 수 있습니다.

Newsgroup이나 Newsgroup 기사의 URL은 보통 페이지의 URL과 비슷하지만 약간 다르기도 합니다.  예를 들면, news:tv.northern-exp라는 URL은 서버 프로토콜(news:) 과 newsgroup의 이름(tv.northern-exp)만 명시 합니다.  보통 페이지의 URL같이 서버 이름과 Pathname을 Slash(/)로 분리 하지 않습니다.  각 Newsgroup의 이름은 URL안에 있기 때문에 유일합니다.

뉴스를 읽는 것은 보통 WWW 페이지들을 읽는 것과 비슷하지만 이로운점와 불리한점도 있습니다.  Usenet는 서버와 서로 장호 작용(interact)할 기능이 있어서 좋습니다: 이용자의 작품을 올리거나 올려진 기사의 작가에게 쉽게 통할 기능.  그러나, WWW페이지같이 이미지나 다른 멀티미디어가 없어서 읽기가 덜 재미있다고 생각할 수도 있습니다.
Usenet에 익숙해짐


각 Newsgroup은 기사들을 Outline포맷으로 정리합니다.  예를 들면, 한 기사를 읽고 대답(response)을 하는 사람이 많으면, 그 기사 밑에 대답들을 목록합니다.  모든 대답들은 들여서 써있기 때문에 최초의 기사하고 구별할 수 있습니다.  대답들도 비슷하게 대답한 것들을 같이 모아여서 정리합니다; 이렇게 같이 모여진 그룹(group)을 Thread라고 합니다.  모든 이용자들이 보내는 기사들을 그대로 올리면 복잡해서 읽기가 어려워지니까 Thread포맷으로 정리합니다.

각 Newsgroup페이지 내에는 몇개의 명령 버튼들이 제공되고 있습니다.그러나, 제공되는 명령들은 newsgroup페이지(다양한 newsgroup이나 기사들을 목록하는 페이지) 이거나 기사 페이지(기사의 내용을 진열하는 페이지)인지에 따라서 틀립니다.
Newsgroup페이지에는 다음과 같은 명령들이 제공됩니다.
   
새 기사 올리기
   
모든 기사들을 읽었다고 표시하기
   
모든 기사들을 진열 시키기
   
새 기사들만 진열 시키기
   
현재 newsgroup에 등록하거나 이미 등록된 newsgroup들에게 연결시키기
   
기사 페이지에는 다음과 같은 명령 버튼들이 있습니다
   
한 Thread안에서나, 많은 Thread 사이에서 항해하기
   
어떤 Thread를 읽었다고 표시하기
   
newsgroup페이지로 돌아가기
   
등록된 newsgroup페이지에 가기
   
기사에 답변을 그냥 올리거나 답변을 올리면서 기사 작성자에게 e-mail을 보내기
   
기사의 대답없이, 기사의 작성자에게 e-mail만 보내고 싶을 경우에는 작가의 이름을 클릭한다.(작가의 이름은 항상 페이지 밑에 Link로 되어있습니다)
   


Netscape로 전자우편을 송신하는 기능이 있습니다 (받은 우편은 아직 Netscape로 읽을 수는 없습니다).  우편의 내용을 개인적으로 물론 탈수 있지만, 버튼 하나를 선택하셔서 페이지의 모든 내용(문서만)을 같이 보낼 수가 있습니다.  뿐만 아니라, 이용자의 컴퓨터에 저장된 어떤 파일이라도 우편에 붙여서 전송할 기능도 있습니다.

전자우편을 보내기로는, File/Mail Document를 선택하십시오.나타나는 Dialog Box 에 정보 입력항목이 다음과 같이 있습니다.
From: 이용자의 e-mail 주소 (저장적으로 입력되었을 것입니다).

Mail To: 수신자의 e-mail 주소.

Post To (Newsgroup): 기사를 올리고 싶은 newsgroup의 이름.

Subject: 우편의 제목.

Attachment: 같이 붙이는 파일명.

우편 내용을 입력하는 항목도 물론 있습니다.이 항목 안에 현재 페이지의 URL도 자동적으로 입력되어있습니다 (혹시 페이지의 내용을 같이 붙일 경우에 필요하기 때문에).



수신자의 e-mail주소를 알 필요가 있습니다.  인터넷 주소들은 보통 user ID후 @를 붙여서 서버 주소를 부칩니다.  예를 들면, Netscape회사로 우편을 보낼 경우에는 이 주소로 보냅니다: "info@netscape.com".Quote Document 버튼을 선택하면 그 페이지의 텍스트를 우편 내용에 포함합니다.  그러나, 최초의 내용과 구별하기위하여 Netscape는 우편의 문서 앞에 ">"로 표시합니다.

Attach 명령은 우편과 파일을 같이 붙여서 전송하는 명령입니다. Attach버튼을 선택하시면 진열되는 Dialog Box에 세개의 선택버튼들이 제공됩니다: Document Text, Document Source와 File (이것은 파일 찾기/"Browse"버튼과 함께 있습니다).

Document Text를 선택하면 현재 페이지의 문서를 그대로 전송하는 것입니다 (">" 없이).  Document Source는 문서 뿐만 아니라 HTML 문서와 같이 전송됩니다.  File 버튼은 PC에 있는 파일을 선택해서 같이 전송하는 것입니다.


Netscape로 페이지의 내용을 컴퓨터의 파일로 저장할 수 있습니다.  페이지를 화면에 받은 후 저장할 수도 있고, 받는 대신에 직접 저장할 수 있습니다.

File/Save As 아이템을 선택해서 Source파일(HTML)이나 텍스트 파일로 저장할 수 있는 옵션이 있습니다.  Source 파일로 저장된 파일은 문서 뿐만 아니라 HTML 기호와 같이 저장됩니다.  텍스트 파일은 그냥 문서만 저장됩니다.  External 이미지들은 저장되지만, In-Line 이미지들(문서 사이에 있는 이미지)은 안됩니다.

페이지를 화면에 받아 들이는 대신에 직접 저장할 기능이 있습니다.  그 페이지의 Link를 마우스로 표시하면서, 마우스의 왼쪽 버튼 대신에 오른쪽 버튼을 누르시면 (Macintosh 이용자들은 마우스 버튼을 매 초 동안 계속 누르십시오) Pop-up 메뉴가 진열됩니다.  그 메뉴의 Save This Link As... 아이템을 선택하면 그 Link가 연결해 주는 파일을 받아 들이지 않고 직접 저장됩니다.  마찬가지로, Link 이미지 파일로 연결해 주면 Pop-up 메뉴의 Save This Image As... 아이템은 이미지를 받아 들이지 않고 직접 저장됩니다.

파일이 하드 디스크(Hard Disk)에 저장된 후, File/Open아이템으로 언제나 Netscape로 불러올 수 있습니다.  이미지 파일을 열 때 에는 "Open File" Dialog Box에 파일 종류(File Type)가 All Files로 설정된 것을 확인 하십시오.

안내: Netscape에 특별한 의미가 있는 문자가 다음과 같이 있습니다.  이 문자들은 파일명 안에 없는 것이 좋습니다.
페이지 인쇄와 단어 찾기


Netscape의 File메뉴의 대부분의 아이템들은 다른 프로그램들의 메뉴 아이템들과 비슷합니다.  다른 프로그램들과 같이, 페이지 내용의 선택된 부분들을 복사하고 인쇄할 수 있습니다.

File/New Window 아이템은 새 Netscape window를 불러옵니다.이 창(window)은 처음 창과 완전히 독립된 창으로 기능을 합니다.  동시에 한꺼번에 일어나는 네트워크 접속 기능이 있습니다.

현재 페이지의 내용을 인쇄하기로는 File/Print아이템이나 Toolbar의 Print아이콘을 선택하십시오.  "Print" Dialog Box가 화면에 나와서 인쇄 옵션들과 인쇄 시작 명령들을 제공합니다.  Windows에서는 인쇄 미리 보기 기능도 있습니다.  File/Print Preview 아이템을 선택하십시오.  인쇄할 때, Netscape가 프린터의 종이 크기에 따라서 파일 내용을 자동적으로 재편성합니다.

Windows 사용자들은 인쇄 옵션들을 설정할때 "Print" Dialog Box의 Print Setup버튼을 선택하십시오.  Macintosh 사용자들은 File/Page Setup아이템을 선택하십시오.  랜드스캐프 인쇄 (종이 옆으로 인쇄) 기능도 제공되어있습니다.

Cut (잘라 내기), Copy (복사) 나 Paste (붙여 넣기)를 하기로는, Edit 메뉴에서 알맞은 아이템을 선택하십시오.  페이지의 내용은 이용자가 보통 변경할 수 없기 때문에 Paste명령은 Netscape 안에서 가끔 사용하지만(예: 정보입력항목), 페이지의 문서를 복사해서 다른 응용 프로그램에 붙여 넣는 경우가 많습니다.

페이지 내용에서 한 단어나 단문을 찾으려면, Edit/Find 아이템이나 Toolbar의 Find아이콘을 선택하십시오.  진열되는 Dialog Box에 찾는 단어나 단문을 입력하십시오.  입력한 단어를 내용 안에 있는 단어하고 Capitals도 맞게하게 위해서는 Match Case라디오 버튼을 선택하십시오.  똑같은 단어/단문을 다시 찾고 싶을 경우에는 Edit/Find Again아이템을 선택하십시오.
