            TCP/IP와 Protocol
Protocol 
            (프로토콜)
컴퓨터와 컴퓨터가 통신을 주고 받을 때 어떠한 방식과 형태로 보내라는 서로의 약속된 통신규약을 프로토콜이라 
            합니다. 
 서로 다른 방식으로 이야기한다면 서로 알아 듣지 못할 것입니다. 그래서 서로 약속된 방식으로 통신하는 
            규약을 만든 것입니다. 인터넷의 대표적인 프로토콜로 TCP/IP가 있습니다.
 프로토콜은 ISO(국제 표준화기구)가 정한 규격에 따라 만들어지며, 7개의 계층으로 구성되어 있습니다. 
            그 7가지의 프로토콜은 IP, TCP, UDP, Telnet, DNS, FTP, SMTP 등입니다. 
 E-mail은 기본적으로 SMTP(Simple Mail Transfer Protocol)를 프로토콜로 
            사용하고 있습니다. 다른 프로토콜로서 POP3(The Post Protocol - verserion3)와 MIME(Mpurpose 
            Internet Mail Extension)이라는 프로토콜도 있습니다. 

              
IP (Internet Protocol) 

              
TCP (Transmission Control Protocol) 

              
UDP (User Datagram Protocol) 

                : 주로 데이터전송을 담당하며 네트워크의 관리를 운영 

              
Telnet DNS (Domain Name Service) 

                : 인터넷상에 위치한 서버들의 주소를 식멸하는 기능, 서버를 식별하기 위한 고유한 이름을 부여

              
FTP (File Transfer Protocol) 

              
SMTP (Simple Mail Transport Protocol) 

                : 전자우편 프로토콜 
TCP/IP(Transmission 
            Control Protocol / Internet Protocol)
많은 나라에서 수많은 이 기종 컴퓨터들을 함께 작업하도록 하기 위한 프로토콜로 인터넷의 대표적인 
            프로토콜입니다. 
 호스트에서 호스트로 전송되는 정보는 패킷(Packet)이라는 적은 단위로 나누어서 전송합니다. 
            이 때 TCP는 데이터의 흐름을 관리하고 데이터가 정확한지 검사하는 역할을 하고 IP는 데이터 패킷을 한 장소에서 다른 
            곳으로 옮기는 역할을 합니다.
 TCP/IP는 정확히 말하면 하나의 프로토콜로 이루어진 것이 아니라 다수의 프로토콜의 집합으로 
            구성되어 있습니다. TCP/IP를 구성하는 프로토콜들은 다음과 같습니다. 

              
 TCP (Transmission Control Protocol) 

              
IP (Internet Protocol) Serial Line Internet Protocol (SLIP) 
                

                : 전화선을 이용하여 TCP/IP를 사용할 때 필요한 프로토콜 

              
Point-to-Point Protocol (PPP) 

                : 전화선을 이용하여 TCP/IP를 사용할 때 필요한 프로토콜로 SLIP와 속도에서 별차이가 없지만 에러탐지 능력이 
                있어서 잡음에 강하고 안정적이지만 약간의 오버헤드가 있습니다. 

              
Internet Control Message Protocol (ICMP) 

              
Interior GatewayProtocol (IGP) 

              
Exterior Gateway Protocol (EGP))
